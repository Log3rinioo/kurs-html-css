Czym jest HTML?

HTML (ang. Hypertext Markup Language) – hipertekstowy język znaczników. Jest to jeden z kilku języków używanych do tworzenia stron internetowych. Służy on do opisu strony, a nie jej wyglądu. W przeciwieństwie do języków programowania, które używają skryptów do wykonywania funkcji, język znaczników używa tagów (inaczej znaczników, czyli wyrażeń w ostrych nawiasach <>) do identyfikacji treści.

Historia języka HTML

Początki języka HTML sięgają roku 1960, gdy IBM pracował nad systemem prezentowania i organizowania dużych ilości danych. Nazwano go wówczas GML (ang. General Markup Language) a po latach prac, w 1978 roku przekształcono na SGML (ang. Standard Generalized Markup Language). SGML to tak zwany metajęzyk, czyli język na podstawie, którego tworzone są języki podrzędne (HTML jest właśnie takim językiem wywodzącym się z SGML).

Twórcą języka HTML jest Timothy Berners-Lee. Jego pierwsza wersja została opracowana w 1989 roku. Tim w tym czasie był fizykiem oraz konsultantem do spraw oprogramowania pracującym w Europejskim Ośrodku Badań Jądrowych CERN (Conseil Européen pour la Recherche Nucléaire) koło Genewy. Pierwotnie język składał się z kilkunastu znaczników umożliwiających wyświetlenie tekstu wraz z odsyłaczami do innych tekstów (dodanie znacznika a). Rewolucyjność pomysłu polegała na tym, że użytkownik przeskakujący do innego tekstu nie musiał w ogóle wiedzieć, gdzie fizycznie znajduje się interesujący go tekst. HTML miał pozwalać autorom na opis struktury dokumentu, wyróżnianie nagłówków, paragrafów, przerw, list i tabel. Sposób prezentacji zależał od programu odczytującego dokument. Była to tzw. zerowa wersja HTML.

Oprócz tego, Tim Berners-Lee stworzył również pierwszą przeglądarkę WWW (o nazwie WorldWideWeb), pierwszy serwer WWW, koncepcję URL i protokół HTTP. Obecnie kieruje Konsorcjum W3 (które zajmuje się ustanawianiem standardów pisania i przesyłu stron WWW).

Wersje języka HTML

  • HTML 2 – listopad 1995
  • HTML 3.0 – kwiecień 1995 r.
  • HTML 3.2 – styczeń 1997 r.
  • HTML 4.0 – styczeń 1997 r.
  • HTML 4.0 (poprawiona wersja) – kwiecień 1998 r.
  • HTML 4.01 – grudzień 1999 r.
  • HTML 5 – październik 2014 r.

Zastosowanie

Język HTML pozwala stworzyć opis strony internetowej, czyli nazwać jej poszczególne elementy. Dla przykładu weźmy artykuł w gazecie. Artykuł ten będzie posiadał zazwyczaj przynajmniej dwa elementy: nagłówek i paragraf. W języku HTML oddzielnie oznaczymy tekst nagłówka, a oddzielnie tekst paragrafu. Samo oznaczenie elementów nic szczególnego nam nie daje. Musimy je jeszcze sformatować przy użyciu języka styli CSS (o tym mowa w drugim kursie).

Edytory tekstu

Aby napisać stronę internetową potrzebujemy skorzystać z jakiegoś edytora tekstu. Dokument HTML jest zwykłym plikiem tekstowym, więc możemy go edytować nawet za pomocą najprostszego edytora tekstu – notatnika w systemie Windows. Metoda taka byłaby jednak dość uciążliwa, dlatego na rynku pojawiło się wiele wyspecjalizowanych edytorów, które ułatwiają konstruowanie dokumentu, wspomagając wprowadzanie poleceń. Poniżej znajdziesz listę niektórych edytorów HTML.

Adobe Dreamweaver CC

Adobe Dreamweaver CC to aplikacja IDE (Integrated Developement Environment), która obsługuje zarówno programowanie back-end, jak i front-end. Ponadto oprogramowanie zapewnia zestawy narzędzi do projektowania i tworzenia stron internetowych, które ułatwiają tworzenie witryn internetowych. Adobe Dreamweaver CC zawiera solidne narzędzie do edycji kodu, które obsługuje różne języki, takie jak HTML, CSS i JavaScript. Ponadto oprogramowanie pozwala użytkownikom wybierać między edytorem tekstowym i WYSIWYG (What You See Is What You Get) lub łączyć oba. Edytor tekstu ma wiele pomocnych funkcji, takich jak podświetlanie składni, uzupełnianie kodu i obsługa wielu języków. Co więcej, edytor wizualny jest wyposażony w funkcję przeciągania i upuszczania. Chociaż nie jest dostępna darmowa wersja, ten edytor stron internetowych oferuje siedmiodniowy okres próbny. Następnie, aby dalej korzystać z oprogramowania, dostępnych jest wiele planów płatności zaczynając od 24,59 € miesięcznie. Kilka ważniejszych funkcji to:

  • Szablony startowe – zapewniają podstawową strukturę strony internetowej, którą użytkownicy mogą dostosować do różnych stron, od blogów po portfolio.
  • Responsywne projekty – układy płynnej siatki automatycznie dostosowują rozmiar elementów witryny do różnych rozmiarów ekranu na różnych urządzeniach.
  • Obsługa Git – wydajnie zarządzaj kodem witryny i wykonuj wiele operacji Git, takich jak „push”, „pull” i „fetch” z pulpitu nawigacyjnego Dreamweaver.
  • Dostęp do bibliotek Creative Cloud — zapewnia dostęp do zasobów z innych programów Adobe obsługujących biblioteki Creative Cloud, w tym Photoshop, Premiere Pro i After Effects.

Program Adobe Dreamweaver CC jest dostępny dla systemów Windows i OS X.

Atom

Atom to darmowy edytor kodu o otwartym kodzie źródłowym. Pozwala na dostosowanie interfejsu poprzez wybranie jednego z gotowych motywów lub stworzenie własnego. Chociaż Atom nie jest dostarczany z edytorem wizualnym, oferuje podgląd na żywo Twojej strony internetowej. Oprócz pisania kodu HTML Atom obsługuje wiele języków programowania, takich jak JavaScript, Node.js i CSS. Dodatkowo można doinstalować funkcję Teletype, która ułatwia współpracę przy projektach z innymi programistami. Inne funkcje Atomu obejmują:

  • Wbudowany menedżer pakietów
  • Wiele okienek – umożliwia podział interfejsu na wiele okien, aby łatwo porównywać i pisać kod w różnych plikach.
  • Podświetlanie składni – ułatwia wykrywanie błędów i identyfikację różnych typów kodu i języków kodowania.
  • Inteligentne autouzupełnianie – pomaga użytkownikom szybciej tworzyć kod dzięki inteligentnym autosugestiom.

Brackets

Brackets to edytor HTML typu open source. Jest bezpłatny i dobrze obsługiwany przez firmę Adobe i społeczność open source. Obsługuje języki HTML, CSS, LESS, SCSS i JavaScript, co czyni go idealnym do programowania front-end. Zawiera również funkcję podglądu na żywo, aby zaoszczędzić czas. Dostępna jest również walidacja kodu W3C, integracja z Git oraz dostępne bezpłatne rozszerzenia z wieloma innymi przydatnymi funkcjami. Brackets zapewniają profesjonalne doświadczenie i są łatwym w użyciu edytorem tekstu dla HTML. Dostępny jest na systemy Windows, OS X oraz Linux.

Notepad++

Notepad++ to darmowy, otwartoźródłowy edytor tekstu. Program jest lekki i ma prosty interfejs użytkownika. Notepad++ jest dystrybuowany jako darmowe oprogramowanie, a jego repozytorium jest dostępne na GitHub. Ten edytor HTML przeznaczony jest wyłącznie na systemy Windows. Elastyczny interfejs użytkownika umożliwia użytkownikom wybór między układami dzielonymi lub pełnoekranowymi podczas kodowania. w układzie podzielonego ekranu użytkownicy mogą pracować jednocześnie nad dwoma różnymi dokumentami. Ten zaawansowany edytor tekstu obsługuje również wiele języków do tworzenia stron internetowych, od HTML i CSS po JavaScript i PHP. Inne najważniejsze funkcje Notepad ++ obejmują:

  • Potężne narzędzia do edycji kodu – jako edytor tekstu oparty na Scintilla, Notepad++ zapewnia dużą szybkość przetwarzania przy mniejszym rozmiarze programu.
  • Integracje wtyczek – rozszerzaj funkcjonalność i dodawaj bardziej zaawansowane funkcje, tworząc nowe wtyczki lub instalując wtyczki firm trzecich z listy.
  • Połączenie FTP – pomaga użytkownikom łączyć się bezpośrednio z plikami serwera i edytować je z pulpitu nawigacyjnego.
  • Mapa dokumentu – wyświetla przegląd i sekcje dokumentu, ułatwiając użytkownikom nawigację w obrębie większego pliku.

Sublime Text

Sublime Text to płatny edytor, ale można go pobrać za darmo na 30 dni. Wyróżnia się tym, że jego obsługa opiera się głównie na klawiaturze (skrótach klawiszowych). Oprócz tego wspiera funkcje jak:

  • Kolorowanie składni
  • Praca na wielu kartach – możliwość przechodzenia między kolejnymi dokumentami nie wychodząc z programu
  • Numerowanie kolejnych linii kodu
  • Multi zaznaczanie – możemy pisać w kilku miejscach na raz
  • Szybkie wyszukiwanie

Visual Studio Code

Visual Studio Code to darmowy edytor HTML typu open source firmy Microsoft, zbudowany na platformie Github Electron. Dzięki tej strukturze użytkownicy mogą tworzyć projekty przy użyciu HTML, CSS i JavaScript na wielu systemach operacyjnych. Visual Studio Code wspiera systemy Windows, Mac i Linux. Integruje się również z Microsoft Azure, co ułatwia wdrażanie kodu. Użytkownicy mogą tworzyć projekty i aplikacje lokalnie oraz publikować je na platformie Azure za pomocą jednego kliknięcia. Ten edytor kodu korzysta z funkcji IntelliSense, aby zapewnić różne typy automatycznego uzupełniania, w tym zmienne, pola i definicje funkcji. Ponadto użytkownicy mogą instalować rozszerzenia językowe, takie jak Python i Ruby, aby umożliwić IntelliSense współpracę z innymi językami programowania. Co więcej, jego przejrzysty i prosty interfejs ułatwia lokalizowanie różnych narzędzi do edycji HTML, otwieranie nowego pliku i wyszukiwanie dokumentów. Istnieje także możliwość uruchomienia VS Code w przeglądarce, klikając tutaj Inne istotne cechy to:

  • Debugowanie – wbudowane narzędzie do debugowania jest dostępne na pulpicie nawigacyjnym, aby szybko edytować, kompilować lub debugować kod.
  • Edytor WYSIWYG (ang. What You See Is What You Get) – po prostu zainstaluj rozszerzenie, aby zmienić domyślny edytor tekstu na edytor HTML WYSIWYG.
  • Fragmenty kodu – zawiera wbudowane fragmenty obsługiwane przez technologię IntelliSense, co ułatwia wprowadzanie powtarzających się wzorców kodu.
  • Użytkownicy mogą jednocześnie pracować nad różnymi projektami lub plikami z różnych folderów nadrzędnych.

Deklaracja

Kod strony internetowej należy rozpocząć od deklaracji oznaczającej, że używamy języka HTML w wersji 5. Aby to uczynić, należy napisać na początku pliku <!DOCTYPE html>. w zależności od wersji, deklaracja ta może wyglądać nieco inaczej.

<html lang="pl">

Chociaż na przestrzeni lat wydano różne wersje, podstawy HTML pozostają takie same. Struktura dokumentu HTML została porównana ze strukturą kanapki. Ponieważ kanapka ma dwie kromki chleba, dokument HTML zawiera otwierające i zamykające znaczniki HTML. Te znaczniki, podobnie jak chleb w kanapce, otaczają wszystko inne. Dodatkowo znacznik <html lang="pl"> może przyjąć atrybut lang, który informuje przeglądarkę o języku stosowanym na naszej stronie.

Komentarze

Niezależnie czy tworzymy niewielką stronę firmową, czy też dużą aplikację internetową, w pewnym momencie nasz kod stanie się dość obszerny. Nawet mała strona internetowa może sięgać kilkuset linijek. Jak się w tym wszystkim połapać? Oczywiście będąc na bieżąco w kodzie, będziemy w stanie spamiętać gdzie co jest. Natomiast jak wrócimy do strony po pewnym czasie gwarantuje, że zaczniemy się głowić nad naszym własnym kodem i próbować rozwiązać co tam też napisaliśmy i po co? z pomocą przychodzą nam komentarze. Jest to ludzki element każdego kodu, niezależnie czy kodujemy w HTML i CSS, czy programujemy w PHP'ie. Komentarze występują w każdym języku programowania, głównym ich zadaniem jest pomoc człowiekowi rozszyfrować o czym piszemy i czego dana część kodu służy, szczególnie, gdy pracujemy w wieloosobowych zespołach nad jednym kodem. Każdy język programowania ma swoją składnię, w języku HTML komentarze zapisujemy pomiędzy znacznikami <!– oraz –>. Wszystko co zostanie zapisane pomiędzy tymi znacznikami zostanie pominięte przez przeglądarkę przy wyświetlaniu treści. Będą one jedynie widoczne dla osób przeglądających kod strony internetowej.

<!DOCTYPE html> <!-- Deklaracja rodzaju dokumentu -->
<html lang="pl"> <!-- Deklaracja języka HTML i polskiego -->

</html> <!-- Zakończenie dokumentu -->

<head>

Zaraz po otwierającym tagu <html lang="pl"> znajdziesz nagłówek dokumentu, który jest identyfikowany przez otwierające i zamykające tagi head. Nagłówek pliku HTML zawiera wszystkie niewizualne elementy, które pomagają w działaniu strony. Takich elementów może być wiele, lecz najczęściej używane są <meta charset="UTF-8">, który informuje przeglądarkę, aby korzystała z kodowania UTF-8 (dzięki czemu możliwe jest rozpoznanie polskich znaków) oraz <title>Tytuł</title>, który określa tytuł strony (przykładowo Kurs HTML | Piotr Winiarczyk 3K).

<!DOCTYPE html>
<html lang="pl">
<head>
<!-- Miejsce na ustawienia dokumentu HTML -->
<meta charset="UTF-8">
<title>Tytuł</title>
</head>
</html>

<body>

Tag <body> następuje po tagu <head>. Wszystkie elementy wizualno-strukturalne są zawarte w tagu <body>. Nagłówki, akapity, listy, cytaty, obrazy i linki to tylko kilka elementów, które mogą znajdować się w tagu <body>. Podstawowa struktura HTML:

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<!-- Miejsce na treść stony -->
<h1>Wpisz imię i nazwisko</h1>
</body>
</html>

Nagłówki

Język HTML zawiera sześć poziomów nagłówków, które są uszeregowane według ważności. Są to <h1>, <h2>, <h3>, <h4>, <h5>, i <h6>. Poniższy kod pokazuje różnice pomiędzy nimi:

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<h1>Nagłówek 1</h1>
<h2>Nagłówek 2</h2>
<h3>Nagłówek 3</h3>
<h4>Nagłówek 4</h4>
<h5>Nagłówek 5</h5>
<h6>Nagłówek 6</h6>
</body>
</html>

Akapity

Aby utworzyć akapit należy użyć otwierającego i zamykającego znacznika <p>, a pomiędzy nimi wstawić treść akapitu. Aby wymusić przejście do następnej linijki należy użyć znacznika <br />.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<p>Oto akapit.</p>
<p>Oto <br />drugi akapit.</p>
</body>
</html>

Formatowanie tekstu

W języku HTML istnieje wiele elementów służących do określenia stylu tekstu. Są to:

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<p>Zwykły tekst w akapicie (dodaje odstępy)</p>
<span>Zwykły tekst poza akapitem</span><br />
<b>Tekst pogrubiony</b><br />
<strong>Tekst pogrubiony</strong><br />
<i>Tekst pochylony</i><br />
<em>Tekst pochylony</em><br />
<small>Tekst pomniejszony</small><br />
<big>Tekst powiększony</big><br />
<mark>Tekst podświetlony</mark><br />
Indeks <sup>górny</sup><br />
Indeks <sub>dolny</sub><br />
<u>Tekst podkreślony</u><br />
<ins>Tekst podkreślony</ins><br />
<s>Tekst przekreślony</s><br />
<del>Tekst przekreślony</del><br />
</body>
</html>

Wielkość tekstu

Aby zmienić rozmiar tekstu, należy do znacznika dodać argument style="font-size: x;", gdzie jako x podajemy rozmiar czcionki wraz z jednostką (np. 16px, 12 pt). Więcej na ten temat dowiesz się tutaj.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<p>Oto domyślny akapit.</p>
<p style="font-size: 26px;">Oto akapit o wielkości 26px.</p>
</body>
</html>

Kolor tekstu

Aby zmienić kolor tekstu, należy do znacznika dodać argument style="color: x;", gdzie jako x podajemy kolor, które można zapisywać poprzez:.

  • nazwę w języku angielskim – green
  • kod szesnastkowy – #008000
  • kod RGB – rgb(0, 128, 0)
  • kod HSL – hsl(120, 100%, 25%)
  • kod HWB – hwb(120 0% 50%)

Kolorów z nazwą w języku angielskim, których można użyć na stronie internetowej jest na ten moment 140, a ich pełną listę znajdziesz tutaj. Natomiast korzystając z innego sposobu zapisania koloru, możemy uzyskać około 16,7 miliona barw.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<p>Oto domyślny akapit.</p>
<p style="color: blue;">Oto niebieski akapit.</p>
<p style="font-size: 22px; color: blue;">Oto niebieski akapit o wielkości 22px.</p>
</body>
</html>

Pozycjonowanie tekstu

Mamy także możliwość, aby podobnie jak w programie Microsoft Word, pozycjonować tekst:

  • do lewej (domyślnie) – left
  • do środka – center
  • do prawej – right
  • wyjustować – justify

W tym celu do znacznika, który oznacza tekst (np. <span>) dodajemy atrybut style="text-align: x";, gdzie w miejsce x wstawiamy left, center, right lub justify.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<h2> Wyrównywanie do lewej </h2>
<p style="text-align: left;">Lorem ipsum dolor sit amet, consectetur adipiscing elit. Vivamus eget nisi quis diam dapibus porta. Aenean aliquet orci ligula, sit amet porta magna laoreet sed. Nam suscipit nulla turpis, id fringilla nulla suscipit eu. In vitae nunc quis quam vulputate egestas.</p>

<hr>
<h2> Wyśrodkowanie </h2>
<p style="text-align: center;">Lorem ipsum dolor sit amet, consectetur adipiscing elit. Vivamus eget nisi quis diam dapibus porta. Aenean aliquet orci ligula, sit amet porta magna laoreet sed. Nam suscipit nulla turpis, id fringilla nulla suscipit eu. In vitae nunc quis quam vulputate egestas.</p>

<hr>
<h2> Wyrównywanie do prawej </h2>

Animowanie tekstu

Do prostej animacji tekstu możemy użyć znacznika <marquee>. Umożliwia on przewijanie tekstu lub obrazu na stronie internetowej od lewej do prawej lub odwrotnie, od góry do dołu lub odwrotnie. Znacznik <marquee> wspiera dużo różnych atrybutów, wśród których są:

  • Typ animacji – behavior
    1. scroll – tekst przesuwa się od prawej do lewej (domyślnie)
    2. alternate – tekst przesuwa się od prawej do lewej, a następnie „odbija się” i powraca
    3. slide – tekst przesunie się od prawej do lewej tylko raz, a później się zatrzyma i pozostanie nieruchomy
  • Kierunek animacji – direction
    1. left – tekst będzie się przesuwał od prawej do lewej (jest to domyślne zachowanie)
    2. right – tekst będzie się przesuwał od lewej do prawej
    3. up – tekst będzie się przesuwał od dołu do góry
    4. down – tekst będzie się przesuwał od góry do dołu
  • Ilość powtórzeń – loop
<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<marquee>Tu wpisz tekst</marquee><hr>
<marquee behavior="alternate" direction="right" loop="2">Tu wpisz tekst</marquee><hr>
<marquee behavior="scroll" direction="up">Tu wpisz tekst</marquee><hr>
<marquee behavior="alternate">Tu wpisz tekst</marquee>
</body>
</html>

Tekst preformatowany

Znacznik <pre> wprowadza tekst preformatowany, czyli napisany czcionką monotypiczną (o stałej szerokości znaku), który uwzględnia dodatkowe spacje, tabulację i znaki końca linii (nie trzeba stosować znaczników <br />) oraz nie jest automatycznie zawijany. Dzięki niemu możesz np. wkleić na stronę WWW tekst, wprost ze zwykłego edytora, bez stosowania dodatkowych znaczników (niestety informacje dotyczące formatowania zostaną pominięte). Należy jednak przy tym pamiętać, aby tekst nie zawierał znaków: "<" oraz ">" (w zamian używaj: &lt; i &gt;).

</head>
<body>
<pre> 
Ten tekst jest preformatowny
dlatego możemy używać
    tabulatora,

</pre>
</body>

Listy uporządkowane

Listy uporządkowane w języku html to takie, których elementy posiadają pewną numerację, stąd też zwyczajowa nazwa lista numerowana. Numeracje w tych listach możemy uzyskać poprzez cyfry, bądź litery. Listy uporządkowane tworzymy przy pomocy elementu <ol> (ang. ordered list – lista uporządkowana). Jednak sam zapis znacznikowy listy uporządkowanej <ol> nie wystarczy, do tej listy musimy jeszcze dodać tzw. elementy listy.

Elementy listy

Elementy listy to po prostu kolejne jej podpunkty. Tworzymy je przy pomocy elementu <li> (ang. list item – element listy). Pomiędzy znacznikami <li> oraz</li> dodajemy kolejne elementy listy.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<ol>
<li>Punkt pierwszy</li>
<li>Punkt drugi</li>
<li>Punkt trzeci</li>
</ol>
</body>
</html>

Element <ol> wskazuje jedynie na rodzaj listy. Wszystkie elementy listy muszą się znaleźć pomiędzy znacznikami <li> oraz </li>.

Zmiana numerowania

Możemy rozpocząć numerowanie od dowolnej litery lub liczby. By to zrobić musimy dodać atrybut start, a jako wartość ustawić numer pierwszego podpunku (gdy np. ustawimy start="3" lista rozpocznie się od liczby 3 lub litery C).

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<ol start="3">
<li>Punkt trzeci</li>
<li>Punkt czwarty</li>
<li>Punkt piąty</li>
</ol>
</body>
</html>

Drugim rodzajem listy w języku html, jest lista nieuporządkowana, zwana też listą wypunktowaną. Elementy tej listy nie są numerowane, a luźno wymienione po tzw. myślnikach. Domyślnie rolę tych myślników przejmują kropki. Listę nieuporządkowaną tworzymy za pomocą elementu <ul> (ang. unordered list – lista nieuporządkowana). Tak jak w liście uporządkowanej elementy listy również tworzymy poprzez element <li>.

Hiperłącza czyli odnośniki służą do wewnętrznego poruszania się po witrynie internetowej, ale także po całym internecie. Do przemieszczania się po podstronach jednej witryny wykorzystujemy hiperłącza wewnętrzne, natomiast do przemieszczania się pomiędzy róznimi witrynami używamy hiperłączy zewnętrznych. Do stworzenia hiperłącza używamy znacznika <a>.

Dlaczego akurat litera a?

Dlaczego znacznik definiujący link na stronie to właśnie ? Prawdopodobnie wzięło się to od angielskiego słowa anchor, które oznacza kotwicę (hiperłącza są – jak to mówimy – zakotwiczone w dokumencie). Element <a> jest złożony z dwóch znaczników – posiada zatem tag otwierający oraz zamykający: <a></a>. Jest to logiczne, gdyż musimy zdefiniować obszar, który stanowi rzeczywistą kotwicę linku, namacalne obiekty możliwe do kliknięcia. Przykładowe hiperłącze prowadzące do Wikipedii:

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<a href="https://pl.wikipedia.org">Wikipedia</a>
</body>
</html>

Atrybut href

Atrybut href to skrót od ang. hypertext reference – określamy tutaj adres dokumentu HTML, do którego hiperłącze ma prowadzić. Reference oznacza z ang. odniesienie – i rzeczywiście czasami tak określamy linki – mówimy, że to odnośniki do innych dokumentów. Co ważne – atrybut href nie jest wcale wymagany. Mogą istnieć znaczniki <a>, bez podania adresu linka – np. w menu głównym strony (w górnej belce witryny). Dochodzimy tutaj do wniosku, iż tak naprawdę hiperłączem możemy nazwać jedynie element <a>, który posiada określony wartością atrybut href – sam element <a> odnośnika jeszcze nie ustanawia.

Atrybut target

Hiperłącze może także posiadać atrybut target (ang. cel), który określa gdzie docelowo w przeglądarce ma trafić podlinkowany dokument. Możliwe wartości atrybutu target są następujące:

  • target="_self" – otwórz stronę w tej samej karcie/ramce , w której znajduje się link (ponieważ jest to zachowanie domyślne, można ten atrybut pominąć),
  • target="_blank" – otwórz witrynę w nowej, nieużywanej karcie przeglądarki (uwaga: nie należy nadużywać tego mechanizmu! Kieruj się empatią wobec internauty i otwieraj nowe karty tylko tam, gdzie rzeczywiście wydaje się to pożądane – w przeciwnym wypadku gość odwiedzający naszą stronę zirytuje się i natychmiast ją opuści),
  • target="_parent", target="_top" – otworzy adres hiperłącza w odpowiedniej ramce – jest to związane z tzw. framesetem (ang. zestaw ramek). Wartość _parent otworzy witrynę w ramce o jeden poziom wyższej we framesetowej hierarchii, a _top w nadrzędnej ramce. Jednak budowanie witryny na ramkach to relikt przeszłości. Są one fatalne zwłaszcza w kontekście SEO (ang. Search Engine Optimization), jak również niewygodne dla internauty. w praktyce więc raczej nie zdarzy Ci się używać tych wartości atrybutu target.
<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<a href="https://pl.wikipedia.org" target="_blank">Wikipedia w nowej karcie</a><br />
<a href="../Formularze 2/index.html" target="_top">Wewnętrzne hiperłącza do generatora memów</a><br />
</body>
</html>

Grafika i inne elementy multimedialne są jednymi z najbardziej charakterystycznych elementów World Wide Web, ubarwiając i uprzyjemniając pracę w Sieci, ale także dostarczając sporej dawki informacji. Zręczne wykorzystywanie grafiki czy dźwięku podczas tworzenia dokumentu internetowego jest umiejętnością niezmiernie przydatną, choć należy pamiętać, że to właśnie te elementy są główną przyczyną ogromnego wzrostu ruchu w sieci i zatykania się łącz. Konieczne jest zatem wybranie złotego środka między estetyką i nośnością informacyjną grafiki z jednej, a twardymi realiami technicznymi z drugiej strony. Pamiętajmy też, że trzeba zachować należny umiar ze względu na estetykę strony - nadmiar multimediów po prostu razi. Autorzy regularnie odwiedzanych stron (zawierających stałe aktualizacje) mogą jednak liczyć na fakt, że dobrze skonstruowane przeglądarki pozwalają wykorzystywać cache, który gromadzi (jednorazowo) elementy graficzne i dźwiękowe, pozwalając przeglądarce sięgać do zasobów na dysku lokalnym, a nie do Sieci, co wydatnie zmniejsza ruch na łączach. z tego też powodu w Internecie zostały przyjęte dwa podstawowe standardy graficzne - GIF i JPG - które charakteryzują się znaczną kompresją, a tym samym relatywnie niewielkimi rozmiarami plików graficznych. Nie przyjął się jeszcze dostatecznie powszechnie chwalony przez specjalistów format PNG (Portable Network Graphics).

Jak wstawić obrazek?

Obraz definiujemy tagiem <img> (ang. image = obrazek). Znacznik <img> jest pojedynczy, a nie podwójny, gdyż obraz jest obiektem – nie trzeba określać gdzie się zaczyna, a gdzie kończy, ponieważ o tym decyduje rozmiar źródłowej grafiki lub ewentualnie określone przez nas właściwości CSS.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<img src="img/html.jpg" alt="HTML to choroba">
</body>
</html>

Atrybut src

Ścieżka dostępu do źródłowej grafiki (z ang. source = źródło). Atrybut src jest wymagany (ang. required), co wydaje się logiczne, gdyż bez źródłowego pliku graficznego src="logo.jpg" ciężko mówić o istnieniu obrazu w dokumencie.

Atrybut alt

Alternatywny, tekstowy opis, który dobrze oddaje czym dana grafika jest, w sensie: co na niej się znajduje (z ang. alternative = alternatywny). Jest to również atrybut wymagany! Dlaczego atrybut alt jest według standardu koniecznie potrzebny? Otóż jeśli grafika załaduje się nam prawidłowo, to w zasadzie można uznać taki opis za zbędny i tylko niepotrzebnie zajmujący miejsce w kodzie. Natomiast pamiętajmy – na naszą stronę trafią także użytkownicy, którzy wyłączyli w swojej przeglądarce ładowanie obrazów, bądź są osobami słabowidzącymi, korzystającymi z czytników witryn. Wówczas taki alternatywny opis, zapewniający słowną reprezentację zawartości obrazka, pomaga im bardziej komfortowo naszą witrynę przeglądać.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<img src="img/bledny-adres.jpg" height="150px" width="100%" alt="HTML to choroba">
</body>
</html>

Zmiana rozmiaru obrazka

W znaczniku <img> możemy także zdefiniować atrybuty określające na sztywno rozmiar obrazu - width i height, czyli odpowiednio szerokość i wysokość.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<img src="img/html.jpg" width="100%" alt="HTML to choroba">
</body>
</html>

Możemy także sami ustawić zarówno szerokość, jak i wysokość, lecz łatwo wtedy jest zmienić proporcje obrazka, przez co go zniekształcić.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<img src="img/html.jpg" height="150px" width="100%" alt="HTML to choroba">
</body>
</html>

Warto do naszej grafiki dodać atrybuj title. Nada on tytuł naszemu obrazkowi, który będzie widoczny po najechaniu kursorem na obrazek.

Obraz niekoniecznie musi być zapisany na naszym dysku. Atrybut src jako wartość może przyjąć link do obrazka na serwerze.

<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<img src="https://c.tenor.com/oVFB1xXQ4FkAAAAC/monke-gorilla.gif" height="200px" alt="Monke Gorilla" title="Monke 🐒">
</body>
</html>

W celu uatrakcyjnienia wyglądu naszej witryny możemy wstawić na nią pliki multimedialne takie jak video lub audio. w tym celu możemy postąpić podobnie, jak z grafiką, wstawiając pliki znajdujące się na naszym serwerze/komputerze lub poprzez link zewnętrzny.

Lokalne pliki wideo

By wstawić plik video znajdujący się na naszym dysku używamy dwóch znaczników. Są to: <video> oraz <source>. Pierwszy z nich jest odpowiedzialny za stworzenie odtwarzacza dla naszego filmu, natomiast drugi wskazuje jaki plik ma zostać odtworzony. Istnieje możliwość użycia kilku znaczników <source>. Dzięki temu, jeżeli pierwszy film z jakiegoś powodu nie zostanie załadowany, uruchomi się następny.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<video width="300px" controls>
<source src="../Multimedia/multimedia/film.mp4">
</video>
</body>
</html>

Pliki wideo z serwisu YouTube

Aby najprościej wstawić film z serwisu YouTube, należy:

  1. Na komputerze otworzyć film lub playlistę, którą chcesz umieścić.
  2. Kliknąć Uostępnij
  3. Z listy opcji udostępniania wybrać Umieść.
  4. Skopiować kod HTML z pola, które się pojawi.
  5. Wkleić go do kodu HTML swojej strony internetowej.
<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<iframe width="300" height="169" src="https://www.youtube.com/embed/GN8EA9mpkdY" title="YouTube video player" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture" allowfullscreen></iframe>
</body>
</html>

Pierwszy z nich jest odpowiedzialny za stworzenie odtwarzacza dla naszego pliku audio, natomiast drugi wskazuje jaki plik ma zostać odtworzony. Tutaj też można wskazać kilka plików znacznikami <source>

Przesyłanie formularza

Aby móc przesłać dane z formularza, musimy dodać jeszcze jeden rodzaj elementu <input>, a mianowicie submit (ang. załącz, wyślij, albo prześlij).

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<form>
<label>Login <input type="text"></label><br /><br />
<label>Email <input type="email"></label><br /><br />
<label>Hasło <input type="password"></label><br /><br />
<input type="submit" value="Prześlij">
</form>
</body>
</html>

Odbieranie danych

W celu wysłania danych naszym formularzem html, musielibyśmy stworzyć plik z instrukcjami np. w języku PHP i zapisać go w folderze z naszą stroną internetową np. pod nazwą form.php. Do elementu <form> dodalibyśmy następnie atrybut action, który wskazywałby ścieżkę dostępu do naszego pliku form.php. Po naciśnięciu klawisza submit akcja z pliku form.php zostałaby wykonana.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<form action="form.php">
<label>Login <input type="text"></label><br /><br />
<label>Email <input type="email"></label><br /><br />
<label>Hasło <input type="password"></label><br /><br />
<input type="submit" value="Prześlij">
</form>
</body>
</html>

Rozróżnianie danych

Dane w obecnej formie nie zostaną właściwie rozpoznane przez serwer, muszą one się od siebie jakoś wyróżniać, tzn. musimy przekazać serwerowi, że ten element musi być zapisany w tym miejscu. Atrybut, który pozwala rozróżnić przesyłane dane to atrybut name. Każde pole musi mieć unikalną wartość atrybutu name. Ze względu na to, że plik form.php nie istnieje, strona po przesłaniu wyrzuci błąd.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<form action="form.php">
<label>Login <input type="text" name="login"></label><br /><br />
<label>Email <input type="email" name="email"></label><br /><br />
<label>Hasło <input type="password" name="haslo"></label><br /><br />
<input type="submit" value="Prześlij">
</form>
</body>
</html>

Grupowanie inputów

Podobne elementy służące do wprowadzania danych można pogrupować za pomocą elementu <fieldset> (ang. field set – zbiór pól). Element ten, nie tylko semantycznie pogrupuje te pola, ale również doda ramkę dookoła. Grupę tą można opisać za pomocą legendy, dosłownie za pomocą elementu <legend>.

<!DOCTYPE html>
<html lang="pl">
<head>
<meta charset="UTF-8">
<title>Wpisz tytuł</title>
</head>
<body>
<form action="form.php">
<fieldset>
<legend>Rejestracja</legend>
<label>Login <input type="text" name="login"></label><br /><br />
<label>Email <input type="email" name="email"></label><br /><br />
<label>Hasło <input type="password" name="haslo"></label><br /><br />
</fieldset><br />
<input type="submit" value="Prześlij">
</form>
</body>
</html>
